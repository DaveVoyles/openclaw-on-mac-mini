🦀 OpenClaw ✓ both

How to run OpenClaw

1 curl -fsSL http://192.168.1.93:8765/install | bash Install once — on LAN. Off-LAN use: https://openclaw.davevoyles.synology.me/install
2 source ~/.zshrc  (or ~/.bashrc) Reload your shell so the OpenClaw launcher is on your PATH
3 OpenClaw Start the interactive terminal chat (capital O, no arguments)
— openclaw "your question" One-shot mode — ask a single question and return to the shell

OpenClaw (capital O) is the interactive launcher. openclaw (lowercase) is the underlying command used for subcommands and one-shot queries. Both are installed by the curl installer above.

Hybrid REPL — in-session slash commands

Available inside OpenClaw / openclaw chat. Natural-language prompts and slash commands coexist; slash commands are handled locally before the LLM sees the input, high-confidence freeform action prompts can auto-route into the matching slash handler, routed extraction is smarter about quoted/fenced shell commands plus append/replace-style edit requests, and high-confidence ordered prompts can decompose into linked plans that auto-run step-by-step with inline progress.

/helpPrint the full in-REPL command reference
/clearReset current conversation history (session stays active)
/quitExit the REPL and return to the shell
/autoroute [on|off]Show or toggle whether high-confidence freeform prompts auto-route to in-REPL actions
/sessionShow compact session summary: ID, plan/task linkage, file count, automation state
/contextShow active cwd, tracked files, linked plan/task IDs, and a bounded preview of the grounding for the next analyze/write/research-style action
/outputsList saved session outputs with 1-based indices
/outputs <index> / /outputs <filename>Preview a saved artifact inline by index or filename
/rollback lastRestore the newest routed safety checkpoint when auto-rollback is available (currently text file edits only)
/events [n]Show recent session events, including structured route records for auto-routed prompts
/plan [<id>|unlink]Show, link, or unlink a plan; reports confirmed vs. validation-unavailable when local validation sources are missing
/task [<id>|unlink]Show, link, or unlink a task; reports confirmed vs. validation-unavailable when local validation sources are missing
/analyze <goal>Run a workspace-aware analysis using the current session context
/research <query>Run the research agent; report is saved into the session outputs
/write <task>Generate a markdown document and save it as a session artifact
/exec [--] <command>Run a shell command with risk-aware approval prompts and session tracking
/edit <path> [--content TEXT] [--append TEXT]Inspect or write a file; shows a unified diff before applying

/outputs lists saved artifacts for the active session using 1-based indices, and /outputs 1 or /outputs recap.md previews the selected artifact inline. /context now includes a bounded preview of the grounding block that the next /analyze, /write, or /research action will inherit. When auto-route is on, high-confidence single-action prompts can jump straight into those same action handlers and OpenClaw prints a short route announcement showing the slash command it picked. Quoted or fenced shell snippets can now route more cleanly into /exec, and explicit append/replace edit wording can route into structured /edit calls when the file target is clear. When a prompt clearly contains ordered actions, OpenClaw can decompose it into a linked plan, attach that plan to the active session, and run each step inline with [step/total] progress. Linked plan, task, and current-step grounding can also sharpen route selection, including explicit references like step 2 or step three when matching plan data is linked to the session. If a prompt is ambiguous, or that grounding is missing, it safely stays in normal chat instead of forcing a tool action. Use /autoroute off to keep every prompt in chat, or /autoroute on to re-enable routing. High/critical routed /exec and /edit steps still follow the same approval flow as typing those commands directly, /rollback last restores the newest recoverable routed edit checkpoint, automatic rollback currently supports text file edits only, exec checkpoints stay manual-recovery only, and only the newest five routed checkpoints are retained. /events records each routed turn as a structured session event. /plan and /task still link immediately, but they now tell you whether local validation confirmed the target or whether validation was unavailable in the current install.
