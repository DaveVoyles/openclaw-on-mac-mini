1. Overview & Architecture

OpenClaw is a Discord bot that acts as an AI-powered operations agent for your Mac Mini Docker stack. It connects to:

  • Google Gemini 2.5 Flash — AI reasoning with function calling (117 skills) — used for tool-requiring, weather, and research queries
  • Ollama gemma4:e4b — local LLM running on the Mac Mini for conversational queries (free, no rate limits, ~15–20 tok/s on M4 Neural Engine)
  • Docker Engine — manage 26+ containers running on the Mac Mini
  • *arr Stack — Sonarr, Radarr, Lidarr, Prowlarr, Bazarr via their APIs
  • Download Clients — SABnzbd (Usenet) and qBittorrent (torrents)
  • Plex / Tautulli — media server monitoring
  • Synology NAS — storage backend

How It Works — Hybrid Routing

You type /ask "hello"
    ↓
OpenClaw checks: does this need tool use? → No
    ↓
Routed to Ollama (local, free, <200ms)
    ↓
You get a fast conversational reply

You type /ask "What's downloading right now?"
    ↓
OpenClaw checks: does this need tool use? → Yes ("download" keyword)
    ↓
Sent to Gemini 2.5 Flash with 117 tool declarations
    ↓
Gemini calls get_download_queue
    ↓
OpenClaw queries SABnzbd + qBittorrent APIs
    ↓
Gemini formats the results into a human-readable answer
    ↓
You get a Discord embed with your active downloads

Gemini can chain up to 5 tool calls per question. Simple chat goes to Ollama at zero cost. The response footer shows which model answered (🔄 auto, 🏠 local, ☁️ gemini).

💡 You can override routing: Use /ask model:local "your question" to force the local model, or /ask model:gemini "your question" to force Gemini. Set a sticky default with /model set.

4. Conversation Threads (Save & Resume)

By default, /ask keeps 30 minutes of context in memory. If you're researching something and want to pick it up later, save the conversation as a named thread. Threads are written to disk and survive container restarts.

Workflow

1. Chat with /ask as normal
   /ask Tell me about the Sonarr upgrade options
   /ask What version is Sonarr on?
   /ask What would break if I upgraded?

2. Save when you want to stop:
   /save sonarr-upgrade
   → ✅ Saved thread sonarr-upgrade (6 messages)

3. Come back later (any time, even after restart):
   /resume sonarr-upgrade
   → ✅ Resumed thread sonarr-upgrade — 6 messages (saved 2026-03-23 14:30)

4. Continue the conversation:
   /ask OK where were we?

Commands

Command What it does
/save <name> Snapshot current conversation to disk. Name: letters, digits, - or _, up to 32 chars.
/resume <name> Load a saved thread and make it the active conversation in this channel.
/threads List all your saved threads with message count and save date.
/forget <name> Permanently delete a saved thread from disk.
💡 Thread names are per-user and stored in /memory/threads/. They are private to you and not visible to other bot users.
💡 Overwrites work: Running /save same-name again replaces the previous save — useful to update a thread you're actively working on.

8. Long-Term Memory (QMD)

OpenClaw has a persistent memory system that survives restarts. Store facts, preferences, IPs, credentials notes, or anything else you want the bot to remember.

/remember <content> [tags]

Store a fact. Tags are optional comma-separated labels for easier recall. 

/remember The NAS IP is 192.168.1.8 tags:network,infrastructure
/remember Sonarr API key rotated on 2026-03-15 tags:sonarr,security
/remember Dave prefers dark mode dashboards tags:preferences
/remember qBittorrent login: admin / check-secrets-env tags:credentials

/recall <query>

Search your stored memories by keyword or tag.

/recall NAS
/recall sonarr
/recall credentials
💡 Tip: The AI can also use memory! When you /ask a question, Gemini can call recall_fact to check if it already knows the answer from stored memories. Try: /ask What do you remember about the NAS?

How It Works Under the Hood

Memories are stored as JSON in /memory/qmd.json. Each entry has a timestamp, content, and tags. Search is case-insensitive keyword matching against both content and tags.

12. Web Search, Real Estate & Git

OpenClaw can extract content from the web, perform deep market research, and interact with Git repositories for code-related tasks.

5-Tier Search Cascade

Every search_web call tries five sources in order until one succeeds:

Tier Method Requires
1 — Perplexity AI AI-powered deep search; returns synthesized answers with citations and source grounding PERPLEXITY_API_KEY
2 — Firecrawl Search + full page content extraction in one API call; returns clean markdown. Handles JS-rendered sites and anti-bot protection FIRECRAWL_API_KEY
3 — Tavily AI-optimized semantic search; returns direct answers, not just links TAVILY_API_KEY
4 — DuckDuckGo Free DDG Lite via free-web-search ClawHub skill None
5 — Bing Lite Scraped with BeautifulSoup4 as last resort None
💡 3-Tier Content Extraction: When browse_url encounters pages that trafilatura can’t parse, OpenClaw tries two fallbacks automatically:
  1. Jina AI Reader — Free service that converts any URL to clean markdown. Handles JS-rendered sites without running a browser. Fast and reliable.
  2. Playwright — Full headless Chromium browser as last resort. Renders the page completely, then extracts text.

🔮 Why Perplexity AI Is the Primary Search Provider

Perplexity is fundamentally different from traditional search engines. Instead of returning a list of links with snippets (like Google or DuckDuckGo), Perplexity reads the sources for you and returns a synthesized answer with inline citations.

FeaturePerplexity (Tier 1)Tavily / DDG (Tiers 3-4)
What you get AI-written answer with cited sources Raw list of search result snippets
Processing needed Ready to use — answer is pre-synthesized Gemini must read all snippets and synthesize an answer
Source quality Google-tier web index with real-time data Good (Tavily) / decent (DDG)
Cost ~$0.005 per query (sonar model) ~$0.01 (Tavily) / Free (DDG)
Token savings Lower — no extra Gemini call to synthesize Higher — raw results + synthesis call

In practice: When you ask “What’s happening at Monday Night Raw tonight?”, Perplexity returns a direct answer like “Monday Night Raw is live tonight from [City] at [Arena]. The card includes [matches].” with source links. Tavily would return 5 search snippets that Gemini then has to piece together — slower, more expensive, less reliable.

Usage tracking: Every Perplexity call is tracked on the Dashboard under the Spending section. You can see total calls, cost, and daily breakdown alongside Gemini API spending.

Setup: Set PERPLEXITY_API_KEY in your .env file. Get a key at perplexity.ai/settings/api. If not set, OpenClaw falls back to Tavily → DuckDuckGo automatically.

For deep multi-source topic research, use /research.

🔥 Integrated Premium Search APIs

Firecrawl — Search + Extract in One Call (Tier 2)

Firecrawl combines web search AND full page content extraction into a single API call, returning clean markdown. It is tier 2 in the search cascade.

  • How it works: Send a search query → get back structured results with full page content already extracted as markdown. No need to browse URLs separately.
  • Why useful: Our research pipeline does search → browse → synthesize (3 steps). Firecrawl collapses the first two into one, making research 30-40% faster.
  • Handles: JS-rendered sites, anti-bot protection, dynamic content.
  • Cost: ~$0.004/page. Free tier: 500 pages/month.
  • Setup: Set FIRECRAWL_API_KEY in your .env file.
  • Website: firecrawl.dev

Serper — Google SERP API (Direct Tool)

Serper provides direct access to Google SERP (Search Engine Results Page) data via API. Installed as a direct tool, not part of the search cascade. Returns the exact same results you’d see on google.com, but as structured JSON.

  • How it works: Send a query → get back Google’s results as JSON (titles, snippets, URLs, knowledge panels, “People Also Ask”, featured snippets).
  • Why useful: Google is still the most comprehensive search engine. Perplexity synthesizes answers (great for Q&A), but sometimes you want the raw Google results for deeper analysis or when Perplexity’s answer is too condensed.
  • Unique data: Knowledge Graph panels, “People Also Ask” questions, shopping results, news results — data other search APIs don’t provide.
  • Cost: $5/month for 2,500 queries. Very affordable.
  • Setup: Set SERPER_API_KEY in your .env file.
  • Website: serper.dev

14. Deep Research (/research)

Multi-step autonomous research. Given a question or topic, the bot:

  1. Plans a set of focused sub-queries using Gemini, generating 2–3 keyword variations per sub-query for broader coverage
  2. Searches each sub-query in parallel (5-tier cascade: Perplexity AI → Tavily → DuckDuckGo → Bing Lite)
  3. Browses the top source URLs to extract full article content (3-tier: trafilatura → Jina AI Reader → Playwright)
  4. Ranks sources by quality tier: academic/official > news > blog > social media
  5. Cross-references claims across multiple sources for accuracy
  6. Synthesizes a structured report using low-temperature Gemini in a new Discord thread, including per-section confidence levels (high/medium/low) and a methodology section

/research <query> [deep:true]

/research Best neighborhoods in Philadelphia under $400k with good schools
/research Docker vs Podman tradeoffs for home lab usage
/research How to set up Tailscale exit nodes on Synology NAS
/research deep:true Current Sonarr v4 migration breaking changes
💡 Deep mode: Add deep:true for extended multi-pass research with exhaustive source coverage. The bot runs additional keyword variation searches and cross-references more sources for higher-confidence findings.
💡 Live progress: A Discord thread is created immediately. The bot posts progress messages as each search and URL is processed. The final report appears with action buttons:
  📌 Save to Memory — stores the report summary in QMD long-term memory
  🔄 Re-run in 24h — schedules the research query to repeat tomorrow
  📎 Save to Vault — writes the full report as a Markdown file to data/vault/Research/

16. Autonomous Agent Operations

OpenClaw includes dedicated tools for long-running, multi-step tasks that require planning, research, and persistent working memory. These tools enable the AI to work autonomously on complex goals (Manus-style) without constant user prompting.

The Planning Workflow

When you give a complex goal via /ask (e.g., "Research and document a full upgrade path for our Docker stack"), the AI uses three persistent files to track its progress:

  1. task_plan.md: The master checklist of phases and steps.
  2. findings.md: Raw data, command outputs, and research notes.
  3. progress.md: A high-level executive summary of current status.

This "working memory on disk" allows the AI to maintain context over long periods and across multiple /ask sessions.

Core Autonomy Tools (via /ask)

  • init_planning_files: Creates the base plan and finding files for a new goal.
  • update_plan_status: Marks phases as complete and updates the current progress note.
  • autonomous_loop: Starts a continuous execution cycle where the AI checks its plan, performs the next action, evaluates the result, and loops until completion.

Controlling Autonomous Loops

Safety is built into the autonomous execution system:

  • Emergency Stop: If a file named STOP_AUTONOMOUS is created in the data/ directory, all loops immediately terminate.
  • Iteration Limit: Loops have a hard-coded maximum number of steps to prevent runaway resource usage.
  • Manual Resume: You can always check progress.md to see where the agent left off and resume the task later.

26. Agent Loop & Plan Management

The Agent Loop is OpenClaw's persistent autonomy engine. When the AI tackles a multi-step task, it creates a plan file (a Markdown .md document) that tracks every step, survives restarts, and lets you monitor progress in real time.

Why Plans?

Without plans, the AI is stateless — if it loses its connection mid-task, all progress is lost. Plans solve this by writing each step to disk as it happens. Think of it as a TODO list the AI manages for itself.

  • Persistent: Plans live as .md files in data/plans/. They survive restarts, crashes, and redeployments.
  • Transparent: You can read any plan file directly (via NAS or /plan-detail) to see exactly what the agent is doing.
  • Resumable: On startup, the bot scans for interrupted plans and notifies you. Use /resume-plan to continue.
  • Human-readable: Plans use standard Markdown with - [x] / - [ ] checkboxes — same as GitHub Issues.

Dependency Tracking

Steps can declare dependencies. For example, “Compare all three” depends on steps 1–3 completing first. The agent won’t start a step until its dependencies are satisfied. This enables future parallel worker execution where independent steps run concurrently.

  • top_headlines(category, country) — Get breaking headlines. Categories: business, technology, sports, entertainment, etc.
  • news_by_source(source_id) — Fetch latest news from specific sources (e.g., "bbc-news", "techcrunch").
// Example questions that trigger news skills:
"What's the latest news about SpaceX?"
"Show me top tech headlines from the past week"
"Any breaking business news today?"

💰 Finance Skills

  • get_market_news(category) — Latest financial news by category (stocks, crypto, forex).
  • get_sentiment_analysis(ticker) — AI sentiment analysis of recent news about a stock.
  • get_box_office_stocks() — Stock performance of major entertainment companies with box office correlation.
  • get_crypto_price(symbol) — Cryptocurrency prices and 24h change.

"What's Tesla's stock price and news sentiment?"
